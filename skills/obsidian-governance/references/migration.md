# Migration Rules

When importing older or external notes:

1. Classify first (note type + destination folder)
2. Move second
3. Rename minimally — fix only vague or duplicate filenames
4. Rewrite only after placement is stable

**Allowed during initial migration:**
- Fix obvious typos
- Remove duplicate `.md` extensions
- Make vague filenames minimally clearer

**Not allowed:**
- Broad renaming across the whole vault without explicit request
- Deleting original material during initial migration

If uncertain → move to `Resources/Import Queue/` for later review.
